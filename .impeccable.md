## Design Context

### Users
- 세일즈팀 팀원 (개발 경험 다양, 비개발자 포함)
- Mac 또는 Windows 사용자
- Claude Code를 처음 접하는 사람
- 셀프서비스로 30분 내 환경 구축이 목표
- Job: 가이드만 따라가면 막힘 없이 동일한 개발 환경 완성

### Brand Personality
따뜻한(Warm), 신뢰감 있는(Trustworthy), 명료한(Clear)

### Aesthetic Direction
- **톤**: Anthropic/Claude 공식 브랜드 느낌. 기술 문서이지만 딱딱하지 않고 따뜻함
- **참조**: docs.anthropic.com (따뜻한 톤, 여백, 명료한 구조)
- **반참조**: 과도한 장식, 화려한 그라데이션, 게이미피케이션 요소
- **테마**: 라이트 모드 기본 + 다크 모드 지원

### Typography

#### Font Stack
- **본문**: 'Pretendard Variable', Pretendard, system-ui, -apple-system, sans-serif
- **코드**: 'JetBrains Mono', ui-monospace, 'SF Mono', 'Cascadia Code', monospace
- **제목**: 본문과 동일 패밀리, font-weight 700

#### Scale
- `--text-xs`: 0.75rem / 1rem (캡션, 배지)
- `--text-sm`: 0.875rem / 1.25rem (부가 설명, 라벨)
- `--text-base`: 1rem / 1.75rem (본문)
- `--text-lg`: 1.125rem / 1.75rem (리드 텍스트)
- `--text-xl`: 1.25rem / 1.75rem (섹션 소제목 h3)
- `--text-2xl`: 1.5rem / 2rem (섹션 제목 h2)
- `--text-3xl`: 1.875rem / 2.25rem (페이지 제목 h1)

### Spacing
Tailwind 기본 스케일 (4px 단위)
- 섹션 간: `py-16` (64px) 또는 `py-20` (80px)
- 섹션 내 블록 간: `space-y-8` (32px)
- 관련 요소 간: `space-y-4` (16px)
- 인라인 간격: `gap-2` (8px) ~ `gap-3` (12px)

### Layout
- 사이드바: 280px 고정 (lg 이상), 모바일에서 오버레이
- 콘텐츠 영역: max-w-3xl (768px)
- 전체 최대 너비: max-w-7xl (1280px)
- 카드/블록 radius: rounded-xl (12px)
- 코드 블록 radius: rounded-lg (8px)

### Components

#### Code Block
- 항상 다크 배경 (라이트/다크 모드 무관)
- 우상단 복사 버튼 (클릭 시 체크 아이콘 전환)
- 좌상단 언어/파일명 라벨
- 가로 스크롤, 줄바꿈 없음
- 패딩: p-4

#### OS Tab (Mac / Windows)
- 탭 바: border-bottom 스타일
- 활성 탭: primary 색상 border-bottom + bold
- 비활성 탭: text-secondary
- 콘텐츠 영역: 탭 바 아래 동일한 카드 내

#### Checklist
- 체크박스 + 라벨 (클릭 시 localStorage 저장)
- 완료 시 line-through + text-secondary
- 상단 진행률 바 (primary 색상)

#### Callout Box (Tip / Warning / Danger)
- 좌측 컬러 보더 (4px) + 아이콘 + 배경색
- Tip: success 계열
- Warning: warning 계열
- Danger: danger 계열
- Info: info 계열

#### Sidebar Navigation
- 섹션별 그룹핑
- 현재 섹션 하이라이트 (IntersectionObserver)
- 클릭 시 smooth scroll
- 모바일: 햄버거 메뉴 -> 오버레이

### Motion
- 페이지 전환 없음 (단일 페이지)
- 호버/포커스: transition-colors duration-200
- 사이드바 모바일 토글: transform translate-x duration-300
- 체크리스트 완료: opacity transition duration-200
- 스크롤 네비게이션: scroll-behavior smooth

### Accessibility (WCAG AA)
- 모든 텍스트 대비 4.5:1 이상
- 인터랙티브 요소 포커스 링 (ring-2 ring-primary/50)
- 키보드 네비게이션 (Tab, Enter, Escape)
- 코드 복사 버튼: aria-label 포함
- 다크 모드 전환: aria-pressed 상태
- 체크리스트: role="checkbox" + aria-checked
- prefers-reduced-motion 대응

### Design Principles
1. **따라하기가 곧 디자인이다**: 시각적 계층이 곧 실행 순서. 위에서 아래로 읽으면 셋업 완료
2. **코드는 주인공**: 코드 블록이 가장 눈에 띄어야 한다. 본문은 코드를 설명하는 보조
3. **OS 분기는 한 눈에**: Mac/Windows 차이를 탭으로 깔끔하게 분리, 혼란 없이
4. **따뜻한 기술 문서**: Anthropic 톤 - 기술적이되 사람 냄새가 나는 문서
5. **진행 감각**: 체크리스트와 섹션 네비게이션으로 "어디까지 했는지" 항상 인지
