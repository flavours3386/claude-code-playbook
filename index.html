02
사전 준비

운영체제에 맞는 패키지 관리자와 기본 도구를 설치합니다.

macOS에서는 Homebrew를 통해 패키지를 관리합니다.

bash
# Homebrew 설치 (패키지 관리자) /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)" # Node.js 설치 brew install node # Git 설치 (Xcode Command Line Tools) xcode-select --install # 설치 확인 node --version # v18 이상 권장 git --version npm --version
✅
Apple Silicon(M1/M2/M3) Mac에서는 설치 후 eval "$(/opt/homebrew/bin/brew shellenv)" 명령을 실행하세요.

Windows에서는 winget 또는 Chocolatey를 사용합니다.

powershell
# winget으로 Node.js 설치 winget install OpenJS.NodeJS # Git 설치 winget install Git.Git # 설치 후 PowerShell 재시작 후 확인 node --version # v18 이상 권장 git --version npm --version
⚠️
winget 설치 후 Node.js가 인식되지 않으면 PowerShell을 완전히 닫고 다시 열어야 합니다.

완료 체크

Node.js v18+ 설치 확인 (node --version)
Git 설치 확인 (git --version)
npm 버전 확인 (npm --version)
03
Claude Code 설치

npm으로 Claude Code CLI를 전역 설치하고 Anthropic 계정으로 로그인합니다.

bash
# Claude Code CLI 전역 설치 npm install -g @anthropic-ai/claude-code # 설치 확인 claude --version # Claude Code 첫 실행 (로그인 프로세스 시작) claude

1

Anthropic 계정 준비

2

claude 명령 실행

환경변수로 설정하려면 ~/.zshrc (Mac) 또는 시스템 환경변수 (Windows)에 추가

bash — API 키 환경변수 설정 (선택)
# Mac: ~/.zshrc 또는 ~/.bashrc에 추가 export ANTHROPIC_API_KEY="YOUR_API_KEY_HERE" source ~/.zshrc # Windows PowerShell: 사용자 환경변수에 추가 [System.Environment]::SetEnvironmentVariable("ANTHROPIC_API_KEY", "YOUR_API_KEY_HERE", "User")

글로벌 설정

Claude Code가 항상 참조하는 전역 지침 파일과 설정을 구성합니다.

프롬프트로 한번에 설정하기

Claude Code가 이미 설치되어 있다면, 아래 프롬프트를 복사해서 붙여넣기만 하면 CLAUDE.md와 settings.json이 자동 생성됩니다.

Claude Code 프롬프트
~/.claude/CLAUDE.md 파일을 아래 내용으로 생성해줘. 이미 있으면 덮어써줘. # Global CLAUDE.md ## 언어 - 모든 응답, 커밋 메시지, 주석은 한국어로 작성 - 코드 내 변수명, 함수명은 영어로 작성 ## 응답 스타일 - 간결하게 핵심만 전달, 불필요한 칭찬이나 서론 생략 - 이모지 사용 금지 (파일명, 커밋 메시지, 응답 포함) ## 작업 원칙 - 작업 시작 전 반드시 프로젝트 구조와 CLAUDE.md를 먼저 파악 - 불확실한 부분은 추측하지 말고 질문할 것 ## 코드 작성 - 기존 코드 스타일과 패턴을 반드시 먼저 파악하고 따를 것 - 새로운 기능 추가 시 기존 코드에 영향이 없는지 확인 - 에러 처리를 반드시 포함 - 코드 중복 금지, 새 함수를 만들기 전에 기존 유틸리티 함수를 먼저 확인하고 재사용할 것 - 매직 넘버/문자열 대신 명명된 상수 사용 - 자기 설명적인 코드를 우선, 주석은 의도가 불명확할 때만 작성 - 부분 구현 금지, 항상 완전한 구현을 할 것 - 사용하지 않는 코드는 삭제할 것 ## 커밋 및 작업 완료 - 기능 단위로 커밋을 나눌 것 (하나의 커밋에 여러 기능 금지) - 작업 중간에도 의미 있는 단위가 완성되면 커밋 - 커밋 메시지 형식: [카테고리] 변경 내용 요약 (카테고리: feat, fix, refactor, docs, style, test, chore) - 작업 완료 시: 프로젝트 CLAUDE.md 업데이트 -> git add -> commit -> push ## 문서 관리 - 새 프로젝트 생성 시 /project-setup 스킬로 표준 구조 스캐폴딩 - 프로젝트 CLAUDE.md에는 항상 최신 상태 유지: 프로젝트 개요, 기술 스택, 주요 명령어, 폴더 구조, 최근 변경사항 - PROGRESS.md가 있으면 진행 상황도 함께 업데이트 - 새 프로젝트 생성 시 워크스페이스 CLAUDE.md의 규칙을 따를 것 ## 테스트 - 기존 테스트가 있는 프로젝트에서는 변경 후 테스트 실행하여 통과 확인 - 기존 테스트가 깨지는 변경은 반드시 사전에 알릴 것 ## 에러 대응 - 원인 분석 -> 해결 방안 제시 -> 수정 순서로 진행 - 동일 에러에 3회 이상 같은 접근으로 실패하면 다른 방법을 시도할 것 - 반복되는 에러는 CLAUDE.md에 트러블슈팅 항목으로 기록 ## 보안 - 시크릿 키, 토큰, API 키 등 민감 정보는 secrets-vault에서 중앙 관리 - 토큰 갱신/재인증 시 secrets-vault의 관련 파일을 최신 상태로 업데이트하고 커밋/푸시할 것 - 토큰 갱신 후 배포 환경의 환경변수도 함께 업데이트할 것 - 프로젝트별 환경변수는 .env 파일로 관리 - 코드에 민감 정보를 절대 하드코딩하지 말 것 - .gitignore에 .secrets-vault, .env 등 민감 파일 포함 확인 - 외부 패키지 추가 시 신뢰성 확인 ## 플러그인 활용 - 설치된 플러그인이 있으면 적극 활용 - 코드 리뷰, 보안 점검, 테스트 관련 플러그인은 작업 완료 전 실행 - 새로운 플러그인이 필요하다고 판단되면 추천 그리고 ~/.claude/settings.json이 없으면 아래 내용으로 생성해줘: { "env": { "CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS": "1" }, "effortLevel": "high" }

Claude Code 프롬프트

MCP 서버 설정

Model Context Protocol(MCP)로 Notion 등 외부 서비스를 Claude Code에 직접 연결합니다.

프롬프트로 설정하기

Notion API 토큰을 준비한 후, Claude Code 안에서 아래 프롬프트를 붙여넣으세요.

Claude Code 프롬프트
~/.claude/.mcp.json 파일을 아래 내용으로 생성해줘. NOTION_TOKEN 값은 내가 알려줄게. { "mcpServers": { "notion": { "command": "npx", "args": ["-y", "@notionhq/notion-mcp-server"], "env": { "NOTION_TOKEN": "여기에_토큰_입력" } } } }
수동으로 설정하기 (참고용)
json — ~/.claude/.mcp.json
{ "mcpServers": { "notion": { "command": "npx", "args": ["-y", "@notionhq/notion-mcp-server"], "env": { "NOTION_TOKEN": "YOUR_NOTION_TOKEN_HERE" } } } }
🔒
보안 주의: Notion 토큰은 절대 코드나 공개 저장소에 하드코딩하지 마세요. .gitignore에 .mcp.json이 포함되어 있는지 반드시 확인하세요.

Notion API 토큰 발급 방법

1

notion.so/my-integrations에 접속

2

"New integration" 클릭 → 이름 입력 → 워크스페이스 선택

3

생성된 "Internal Integration Secret" 복사 → .mcp.json에 붙여넣기

4

Notion 페이지에서 우상단 "..." → "Connect to" → 생성한 인테그레이션 선택

~/.claude/.mcp.json 파일 생성 완료

Step 2. 설치 명령어 실행

Claude Code 올인원 셋업 프롬프트
아래 작업을 순서대로 진행해줘. 각 단계가 완료되면 다음 단계로 넘어가고, 에러가 나면 해결 방법을 알려줘. ## 1단계: 글로벌 CLAUDE.md 생성 ~/.claude/CLAUDE.md 파일을 아래 내용으로 생성해줘: # Global CLAUDE.md ## 언어 - 모든 응답, 커밋 메시지, 주석은 한국어로 작성 - 코드 내 변수명, 함수명은 영어로 작성 ## 응답 스타일 - 간결하게 핵심만 전달, 불필요한 칭찬이나 서론 생략 - 이모지 사용 금지 (파일명, 커밋 메시지, 응답 포함) ## 작업 원칙 - 작업 시작 전 반드시 프로젝트 구조와 CLAUDE.md를 먼저 파악 - 불확실한 부분은 추측하지 말고 질문할 것 ## 코드 작성 - 기존 코드 스타일과 패턴을 반드시 먼저 파악하고 따를 것 - 새로운 기능 추가 시 기존 코드에 영향이 없는지 확인 - 에러 처리를 반드시 포함 - 코드 중복 금지, 새 함수를 만들기 전에 기존 유틸리티 함수를 먼저 확인하고 재사용할 것 - 매직 넘버/문자열 대신 명명된 상수 사용 - 자기 설명적인 코드를 우선, 주석은 의도가 불명확할 때만 작성 - 부분 구현 금지, 항상 완전한 구현을 할 것 - 사용하지 않는 코드는 삭제할 것 ## 커밋 및 작업 완료 - 기능 단위로 커밋을 나눌 것 - 커밋 메시지 형식: [카테고리] 변경 내용 요약 (카테고리: feat, fix, refactor, docs, style, test, chore) - 작업 완료 시: 프로젝트 CLAUDE.md 업데이트 -> git add -> commit -> push ## 문서 관리 - 프로젝트 CLAUDE.md에는 항상 최신 상태 유지 - PROGRESS.md가 있으면 진행 상황도 함께 업데이트 ## 테스트 - 변경 후 테스트 실행하여 통과 확인 - 기존 테스트가 깨지는 변경은 반드시 사전에 알릴 것 ## 에러 대응 - 원인 분석 -> 해결 방안 제시 -> 수정 순서로 진행 - 동일 에러에 3회 이상 같은 접근으로 실패하면 다른 방법 시도 ## 보안 - 민감 정보는 .env 파일로 관리, 코드에 하드코딩 금지 - .gitignore에 .env 등 민감 파일 포함 확인 ## 2단계: settings.json 생성 ~/.claude/settings.json 파일이 없으면 아래 내용으로 생성해줘: { "env": { "CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS": "1" }, "effortLevel": "high" } ## 3단계: OMC 설치 터미널에서 다음 명령을 실행해줘: npm install -g oh-my-claude-sisyphus@latest 설치가 완료되면 /omc-setup 을 실행해줘. ## 4단계: 완료 확인 위 작업이 모두 끝나면 아래를 확인해서 알려줘: - ~/.claude/CLAUDE.md 파일이 존재하는지 - ~/.claude/settings.json 파일이 존재하는지 - ~/.claude/.omc-version.json 파일이 존재하는지 (OMC 설치 확인)
